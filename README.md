# Projet_PAD_DDR

## Description

Le projet **Projet_PAD_DDR** vise à développer un pad de jeu personnalisé, intégrant à la fois les composants matériels (hardware) et logiciels (software) nécessaires à sa réalisation. Ce pad est conçu pour être utilisé dans des jeux de rythme ou d'autres applications interactives, offrant une expérience de jeu immersive et personnalisable.

## Objectifs

- Concevoir et assembler le hardware du pad, incluant des boutons et interfaces de connexion.
- Développer le software pour gérer les entrées utilisateur, traiter les signaux et communiquer avec des appareils externes (comme un ordinateur ou une console de jeu).

## Cahier des Charges

### Fonctionnalités Requises
- **Hardware** :
    - Boutons résistants et sensibles pour une détection précise des pressions.
    - Interface de connexion USB pour une intégration facile.
    - Alimentation via câble.
- **Software** :
    - Gestion des entrées en temps réel.
    - Compatibilité avec des systèmes d'exploitation courants (Windows, Linux).

### Contraintes
- Facilité d'assemblage et de maintenance.
- Respect de l'environnement.

## Structure du Projet

- **Hardware** : Schémas et composants.
- **Software** : Code source, bibliothèques et exemples d'utilisation.
- **Modèles 3D** : Ensemble des modèles d'interrupteurs pour permettre un ajustement personnalisé.

## Installation et Utilisation

1. **Matériel** : Créer le PCB présent dans le dossier `Kicad`. (V2 : connexion USB Micro, V3 : connexion USB-C)
2. **PAD** : Suivre les différents schémas et utiliser les différents modèles pour les interrupteurs dans le dossier `Model`.
3. **Logiciel** : Cloner le dépôt et compiler le code selon les instructions dans le dossier `Cube Workspace`.
4. **Test** : Connecter le pad à votre appareil et tester les fonctionnalités.
